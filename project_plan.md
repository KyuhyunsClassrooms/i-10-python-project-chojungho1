# 프로젝트 계획서

## 1. 프로젝트 주제

내가 만들 프로그램의 주제: 목적에 따른 식단 제시

예) 기분과 시간에 따른 활동 추천기

---

## 2. 문제 상황

이 프로그램이 해결하려는 문제를 설명하세요.

예) 사용자가 현재 기분과 사용할 수 있는 시간을 입력하면, 조건에 맞는 활동을 추천해 주고 싶다.

내 설명:사람들이 건강 관리나 운동을 할 때 자신의 목적(다이어트, 벌크업 등)에 맞는 식단을 구성하는 데 어려움을 겪는다. 각 식품의 영양 성분(탄수화물, 단백질, 지방)을 일일이 확인하고 계산하기 번거롭기 때문에, 사용자가 선택한 운동 목적에 맞춰 필요한 영양소가 풍부한 음식을 자동으로 필터링하고 추천해 주는 프로그램을 만들고자 한다.


---

## 3. 입력-처리-출력 구조

### 입력

사용자가 입력하거나 프로그램이 사용할 데이터는 무엇인가?

- 가 식품 영양 성분 데이터
- 사용자의 식단 목적 번호 입력
- 

### 처리

입력된 데이터를 어떤 규칙이나 알고리즘으로 처리할 것인가?

- isplay_food_list 함수를 통해 전체 식품 리스트와 영양 성분을 화면에 출력함.
- 사용자가 '1'을 입력하면 각 음식의 단백질 함량을 검사하여 20g 이상인 음식만 추출함
- 사용자가 '2'을 입력하면 각 음식의 탄수화물 함량을 검사하여 30g 이상인 음식만 추출함.

### 출력

프로그램이 최종적으로 무엇을 출력하는가?

- 가상 식품 영양 성분 표 전체 목록
- 사용자의 선택 목적에 맞는 맞춤형 가상 추천 식단 결과 (음식명, 탄수화물, 단백질 함량)
-잘못된 입력 시 안내 메시지 출력

## 4. 2차원 리스트 설계

내 프로그램에서 사용할 2차원 리스트의 이름:food_data

예) foods, movies, characters, questions, records

```python
# 예시 형식
```

### 한 행의 의미
하나의 식품에 대한 이름과 세부 영양 성분 정보 (칼로리, 탄수화물, 단백질, 지방)

| 열 번호 | 의미 | 예시 |

| 2번 열 |탄수화물 함량 (정수, g 단위)|0, 32|
| 3번 열 | 단백질 함량 (정수, g 단위)|31, 1|
| 4번 열 |지방 함량 (정수, g 단위)|3, 0|

## 6. 사용할 제어구조

### 조건문을 사용할 부분

예) 사용자의 예산보다 가격이 낮은 항목만 추천한다.

내 계획:사용자가 입력한 값(choice)이 '1'인지 '2'인지 판별하고, 그에 따라 각 음식의 단백질이 20g 이상인지 혹은 탄수화물이 30g 이상인지 비교 연산자(>=)를 사용하여 조건에 맞는 데이터만 리스트에 추가한다. 결과가 없을 때를 대비한 if not 예외 처리 조건문도 사용한다.


### 반복문을 사용할 부분

예) 2차원 리스트의 모든 항목을 하나씩 검사한다.

내 계획:for food in foods:와 같이 2차원 리스트의 내부 요소를 처음부터 끝까지 하나씩 순회하며 전체 목록을 출력하거나, 조건에 맞는 음식을 필터링(검사)할 때 반복문을 사용한다.

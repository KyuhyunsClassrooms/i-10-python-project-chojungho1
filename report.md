# 최종 보고서

## 1. 프로그램 주제와 목적

내 프로그램의 주제:영양소 분석 및 맞춤형 가상 식단 권장 프로그램

이 프로그램을 만든 이유:건강 관리나 운동 목적(다이어트, 벌크업 등)에 따라 챙겨 먹어야 하는 핵심 영양소가 다릅니다. 하지만 매번 음식의 영양 성분 표를 직접 확인하는 것은 번거롭습니다. 사용자가 자신의 목적을 입력하면 조건에 맞는 최적의 음식을 자동으로 분류하고 추천해 줌으로써, 쉽고 효율적으로 식단을 계획할 수 있도록 돕기 위해 이 프로그램을 만들었습니다.


---

## 2. 문제 분해 및 모델링

### 입력
미리 저장된 식품들의 영양 성분 데이터 (2차원 리스트)

사용자가 선택한 식단 목적 번호 (문자열 '1' 또는 '2')

### 처리
전체 식품 리스트를 화면에 보기 좋게 출력하는 처리

사용자의 선택이 '1'일 때 단백질이 20g 이상인 음식을 필터링하는 처리

사용자의 선택이 '2'일 때 탄수화물이 30g 이상인 음식을 필터링하는 처리

### 출력
가상 식품 영양 성분 표 전체 목록 (텍스트)

필터링 과정을 거쳐 선택된 맞춤형 추천 식단 음식 리스트 및 해당 음식의 탄수화물, 단백질 함량

---

## 3. 사용한 2차원 리스트 설명

2차원 리스트 이름:food_data

한 행의 의미:하나의 가상 식품이 가진 고유한 영양 정보 한 세트

각 열의 의미:

| 열 번호 | 의미 | 실제 예시 |
|---:|---|---|
| 0번 열 |  음식 이름 (문자열)|'닭가슴살', '고구마'|
| 1번 열 | 칼로리 (정수, kcal)|165, 128|
| 2번 열 | 탄수화물 함량 (정수, g)	|0, 32|
| 3번 열 | 단백질 함량 (정수, g)| 31, 1|
| 4 번 열| 지방 함량 (정수, g)| 3, 0|
이 2차원 리스트를 사용한 이유:하나의 음식에 이름, 칼로리, 탄수화물, 단백질, 지방 등 서로 다른 유형의 연관된 데이터들이 포함되어 있기 때문입니다. 이를 하나의 표(행과 열) 형태로 묶어서 관리하고, 인덱스 번호(0~4)를 통해 필요한 영양소만 쏙쏙 뽑아 비교하기에 가장 적합한 구조라 판단하여 사용했습니다.

## 4. 함수 설명

| 함수 이름 | 역할 | 입력값 | 반환값 또는 출력 |
|---|---|---|---|
| display_food_list| 전체 음식의 영양 성분 표를 정렬하여 출력| foods (2차원 리스트)| 없음 (화면 출력)|
|filter_menu| 사용자의 목적에 맞는 음식을 골라내어 저장| foods (2차원 리스트), choice (문자열)| selected_foods (필터링된 리스트)|

가장 중요한 함수 하나를 골라 자세히 설명하세요.

함수 이름:filter_menu

설명:프로그램의 핵심 알고리즘이 담긴 함수입니다. 전체 음식 데이터(foods)를 입력받아 내부에서 for문으로 순회하며 각 음식의 탄수화물(food[2])과 단백질(food[3]) 수치를 검사합니다. 사용자가 선택한 값(choice)이 '1'이면 단백질 20g 이상, '2'이면 탄수화물 30g 이상이라는 조건을 만족하는 음식만 빈 리스트(selected_foods)에 더해준 뒤 최종적으로 이 리스트를 반환(return)합니다.

## 5. 조건문과 반복문 활용

### 조건문 사용 부분

어떤 조건을 검사했는가?

사용자가 입력한 목적 번호가 '1'인지 '2'인지 검사하였고, 각 음식의 단백질 함량이 20 이상(protein >= 20)인지, 탄수화물 함량이 30 이상(carbs >= 30)인지 검사했습니다. 또한 최종 추천 리스트가 비어있는지(if not recommend_recipe)도 검사했습니다.
### 반복문 사용 부분

무엇을 반복 처리했는가?
2차원 리스트인 food_data에 들어있는 모든 음식 요소를 하나씩 꺼내어 화면에 양식에 맞춰 출력하는 작업과, 모든 음식을 순서대로 돌며 영양소 기준 값을 충족하는지 검사하는 작업을 반복 처리했습니다.

### 조건문과 반복문이 함께 작동한 부분
filter_menu 함수 내부에서 for food in foods: 반복문을 통해 전체 음식 데이터를 하나씩 조회하는 도중에, 그 내부에 중첩된 if-elif 조건문을 실행하여 해당 음식을 추천 리스트에 추가할지 말지 결정하는 필터링 알고리즘 구조에서 함께 작동했습니다.

## 7. 테스트와 오류 수정

### 테스트 1: 정상 사례

입력:목적 번호 2

예상 결과:탄수화물이 30g 이상인 '고구마(32g)', '현미밥(33g)'이 출력되어야 함.

실제 결과:예상대로 고구마와 현미밥만 정확히 화면에 출력됨.


### 테스트 2: 예외 또는 경계 사례

입력:목적 번호 3 (잘못된 입력)

예상 결과:안내 메시지 [알림] 올바른 번호를 입력하지 않았거나... 가 출력되어야 함.

실제 결과:예상대로 알림 문구가 출력되며 프로그램이 안전하게 종료됨.


### 발견한 오류와 수정한 방법
원래 코드의 결과 출력 else 문 안에서 식단 결과를 한 줄 출력할 때마다 전체 식품 리스트를 보여주는 display_food_list(food_data) 함수가 불필요하게 반복 실행되는 오류(들여쓰기 미스)가 있었습니다. 반복문 내부의 들여쓰기를 제거하여 한 번만 깔끔하게 출력되도록 위치를 수정했습니다.

## 8. Gemini 활용 내용 요약

Gemini에게 도움받은 부분:작성한 코드의 전체적인 실행 흐름을 주석처럼 쉽게 해석받았으며, 들여쓰기 위치에 따른 논리적 오류가 발생할 수 있다는 피드백을 받아 코드를 보완했습니다


Gemini가 제안했지만 내가 수정하거나 사용하지 않은 부분:무한 루프(while True)를 통한 재입력 유도 코드를 제안받았으나, 과제 조건의 심플함을 유지하기 위해 우선 현재 조건문 단계에서 예외 메시지를 띄우는 방식으로 직접 판단하여 처리했습니다.


최종 코드에서 내가 직접 판단하거나 수정한 부분:결과 출력 창의 들여쓰기 오류를 확인하고 이를 코드 하단부로 완전히 빼내어 가독성을 높이는 수정을 직접 진행했습니다

## 9. 한계와 개선 방안

현재 프로그램의 한계:현재는 저장된 음식의 양이 적고, 사용자가 선택한 단 하나의 음식 기준(20g 이상, 30g 이상)만 체크하기 때문에 개인의 몸무게나 하루 권장 섭취량에 따른 세밀한 계산이 불가능합니다.


다시 개선한다면 추가하고 싶은 기능:사용자의 몸무게를 입력받아 하루 필요 칼로리를 계산해 주고, 추천된 식단들의 '총칼로리 합산 기능' 및 '지방 함량 제한 필터'를 추가하여 더 정밀한 헬스케어 프로그램을 만들고 싶습니다.
